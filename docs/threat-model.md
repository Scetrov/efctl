# Threat model for efctl

## Scope

This document captures the security-relevant assets, trust boundaries, and threats for the `efctl` CLI tool and its supporting repository automation.

## Assets

- Source code (Go), especially command entrypoints and plugin-loading logic.
- Release binaries published via GitHub Releases.
- GitHub Actions workflows and the credentials they use (notably `GITHUB_TOKEN` and any OIDC tokens).
- User configuration and state files consumed or generated by `efctl`.
- Supply-chain artifacts: `go.mod`/`go.sum`, container images, GitHub Actions, and third-party dependencies.

## Trust boundaries

- Developer workstation ↔ GitHub repository (source control).
- GitHub Actions runner ↔ GitHub API / cloud services.
- Upstream dependencies ↔ maintained Go module graph.
- Release artifacts ↔ end users downloading `efctl`.

## Threats and mitigations

| Threat | Description | Mitigation |
|--------|-------------|------------|
| Tampered source | Unauthorized commits to `main` bypassing review | Branch protection, required PRs, CODEOWNERS, signed commits. |
| Malicious dependency | Compromised Go module or GitHub Action in the supply chain | Pinned Action SHAs, Dependabot version/security updates, dependency review, SBOMs. |
| Credential leakage | Secrets accidentally committed or logged | Secret scanning, push protection, short-lived OIDC tokens, least-privilege `GITHUB_TOKEN`. |
| Compromised release | Attacker publishes modified binaries | Signed releases, SLSA/Sigstore provenance, artifact attestations, checksums. |
| Unsafe CI practices | Injection via untrusted context data | Minimal workflow permissions, safe `pull_request_target` handling, CodeQL/SAST. |
| Unreviewed AI/agent changes | Generated code or workflow changes merged without human review | CODEOWNERS review, branch protection, explicit agent credential scope. |

## Dependencies and verification

See `go.mod`/`go.sum` for the Go dependency graph. The project uses Dependabot (`.github/dependabot.yml`) and dependency review workflows to flag vulnerable or unexpected changes.
